\# **13.Базовая архитектура с учётом ограничений бизнес-требований, НФТ, выбранной архитектуры, адресация атрибутов качества.**



\## **13.1 Базовые архитектурные блоки**

|**Компонент**|**Технология**|**Назначение**|
|-|-|-|
|Client|iOS (Swift/HealthKit), Android (Kotlin/Google Fit), Web (React)|Мобильные и веб-клиенты|
|API Gateway|Kong / Traefik|Маршрутизация, rate limit, JWT, логи|
|Модульный монолит|Python / FastAPI|Логика тренировок, соцсети, геймификации, инвентаря, промо|
|Брокер событий|Apache Kafka (KRaft)|Асинхронные события |
|БД транзакционная|PostgreSQL (TimescaleDB для временных рядов)|Тренировки, пользователи, группы, инвентарь|
|Кеш + геоданные|Redis + RediSearch + Geohash|Активные сессии, геопозиции, rate limit счётчики|
|Аналитика|ClickHouse|Агрегаты по тренировкам, региональные сравнения|
|Логи + метрики + трейсы|FluentD → Kafka → ES + Prometheus + Jaeger|Наблюдаемость|
|Объектное хранилище|S3 / MinIO|GPX-треки, фото, аватарки|
|Push-уведомления|Firebase FCM (Android) + APNS (iOS) + Email (SES)|Уведомления|







**## 13.2 Учёт ограничений из требований**

|**Ограничение / требование**|**Как учтено**|
|-|-|
|Пользователи по всему миру|API Gateway в 3 регионах (EU, US, Asia), данные реплицируются асинхронно|
|Соревнования 10K+ участников|Kafka + ClickHouse + шардирование групп, асинхронный расчёт рейтинга|
|Подключение сторонних устройств|Адаптеры в Device Integration Module, универсальный формат TrainingData|
|Интеграция с фитнес-функциями телефона|HealthKit / Google Fit через native SDK|
|Охрана пользовательских данных|Шифрование PII, маскирование в логах, GDPR-согласия, аудит доступа|
|Лёгкая интеграция существующих приложений|Shop Adapter через REST + API Gateway|







**## 13.3 Адресация атрибутов качества**

|**Атрибут**|**Архитектурное решение**|
|-|-|
|Масштабируемость|Горизонтальное масштабирование монолита, Kafka, шардирование PostgreSQL (Citus)|
|Производительность|Кеш Redis, ClickHouse для агрегатов, индексы в PostgreSQL|
|Надёжность|Резервирование БД (primary + replica), Kafka с retention 7 дней, idempotency|
|Доступность|Multi-AZ в каждом регионе, API Gateway с health checks, 99.95%|
|Безопасность|JWT, шифрование, маскирование PII, audit logs|
|Наблюдаемость|Логи (ELK), метрики (Prometheus), трейсы (Jaeger), SLO алерты|
|Расширяемость|Модульная структура, адаптеры для устройств, событийная шина|
