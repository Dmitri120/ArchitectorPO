\# Артефакт 7: Критические бизнес-сценарии



\## Содержание



\### Сценарий 1. Запись и сохранение тренировки

**Актёр:** пользователь с часами/телефоном

**Предусловие:** пользователь авторизован, устройство подключено

**Стимул**: завершение тренировки на устройстве

**Основной поток**:

1. Устройство отправляет данные (дистанция, пульс, темп, GPS-трек)
2. Система сохраняет тренировку в PostgreSQL
3. Система сравнивает с последними 5 тренировками
4. Пользователь видит график "прогресс за месяц"
5. При рекорде → генерируется событие в Kafka → награда + уведомление друзьям

**Критичность**: без этого сценария приложение не имеет смысла.





\### Сценарий 2. Поиск и создание групповой тренировки

**Актёр**: пользователь

**Предусловие**: включена геолокация

**Стимул**: нажатие "найти людей рядом"

**Основной поток**:

1. Система получает текущие координаты (широта/долгота)
2. Поиск в Redis (Geohash) активных пользователей в радиусе 1 км за последние 10 минут
3. Формирование списка с аватарками и видами спорта
4. Пользователь создаёт событие "бег в парке через 15 минут"
5. Push-уведомления найденным пользователям

**Критичность**: основная социальная фича, без неё нет "живого сообщества".





\### Сценарий 3. Сравнение с профессионалом / регионом

**Актёр**: пользователь

**Стимул**: открытие вкладки "Сравнение"

**Основной поток**:

1. Система агрегирует тренировки пользователя за 7 дней
2. Запрашивает агрегированные данные по региону (ClickHouse) — средний темп, дистанцию
3. Запрашивает эталон профи (отмеченные аккаунты)
4. Отображает: "Вы быстрее 60% бегунов в Москве", "Ваш темп на 15% медленнее профи"

**Критичность**: ключевой мотиватор для продвинутых пользователей.





\### Сценарий 4. Получение промоакции после тренировки

**Актёр**: пользователь

**Стимул**: завершение тренировки с определённым видом спорта

**Основной поток**:

1. Система определяет вид спорта (бег, вело, йога)
2. Promotion Service проверяет активные акции в регионе пользователя
3. Если есть совпадение → отправляет push "Скидка 20% на новые кроссовки Nike"
4. Пользователь переходит в магазин через deep link

**Критичность**: связь приложения с коммерческой целью бизнеса.
